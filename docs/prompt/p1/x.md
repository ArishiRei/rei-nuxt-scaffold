# 开发约定 (Conventions)

### 初始化项目

在收到 "初始化项目" 指令后根据项目需求, 设计：

- 项目设计蓝图 **docs/design.md**
- 项目结构注册表 **docs/structure.md**
- 项目任务分配 **docs/task.md**

### 一切架构变更前必须先在 docs/structure.md 登记，再在代码中实现。

交织开发 (Interwoven Development): 
组件化：不写巨型组件，UI 与逻辑分离，通用 UI 沉淀到 src/components。
工具化：重复逻辑必须提取为 src/utils 或 src/composables。
配置化：魔法值、全局开关应提取到 src/app 或环境变量。

### 在开始实现任何功能前，必须先确认项目结构与跨领域约束，否则严禁动手编码。

### 当你接收到任务时，请按照 "Locate -> Analyze -> Implement -> Update" 的循环进行：

- Locate (定位): 在结构文档中找到相关层级（是改页面？还是改工具？）。
- Analyze (分析): 仔细阅读相关层级的内容，理解其功能和作用。
- Implement (实现): 根据分析结果，编写代码实现任务。
- Update (更新): 在本文档中更新相关内容，确保与实现的代码保持一致。
